# Personal Overlay

## Purpose

This overlay adapts the core framework for:
- personal projects
- writing
- learning
- side ventures
- prototyping
- coding
- experimentation

It should keep the system useful without making it heavy.

## What belongs here

- fast iteration
- lightweight templates
- broader experimentation
- looser capability access
- low-friction planning
- practical personal guardrails
- optional use of MCP for expanded capabilities

## What does not belong here

- formal approval chains
- enterprise governance detail
- team audit requirements
- unnecessary process overhead

## Personal operating stance

Use the framework to:
- make work more repeatable
- reduce blank-page friction
- sharpen planning
- improve quality through lightweight review
- capture reusable patterns

Do not use it to:
- automate major life judgement blindly
- outsource important financial or legal decisions
- add complexity for its own sake

## Recommended style

- keep briefs short
- keep plans practical
- use modules selectively
- run review and verification lightly but honestly
- capture patterns only when they recur

## MCP note

In personal use, MCP can be used more freely for experimentation.

Still apply basic discipline:
- know what capability you are giving access to
- know what data is involved
- do not connect tools casually without understanding the implications

## Good default

For most personal tasks:
- clarify
- plan
- execute
- review
- deliver
- capture any reusable learning

That is enough.
