# Weekly Operating Rhythm

Use this rhythm to make the framework habitual without making it heavy.

## Monday: define
- capture active tasks
- clarify the top 3 priorities
- classify each as personal or professional
- create or update briefs and plans

## Tuesday: build
- execute the highest-value task
- use Planner, Researcher, and Implementer as needed
- keep notes on assumptions and blockers

## Wednesday: review
- run Reviewer on active outputs
- tighten structure, logic, and completeness
- identify what needs evidence or approval

## Thursday: verify and deliver
- run Verifier on anything close to complete
- complete delivery notes
- seek human approval where required
- send, publish, or hand over completed work

## Friday: capture and improve
- document what worked
- record one new pattern
- identify one candidate for future automation
- remove anything that added complexity without value

## General rule
Do not try to operationalise everything at once.

Use the framework on real work, then refine based on what repeatedly helps.
