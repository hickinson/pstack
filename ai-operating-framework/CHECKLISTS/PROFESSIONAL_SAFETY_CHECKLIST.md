# Professional Safety Checklist

Use this before delivering professional outputs or using professional tools.

## Data
- Does this involve sensitive or restricted data?
- Should data be minimised?
- Is the data source approved?

## Tooling
- Am I using approved tools and capabilities?
- If using integrations, are they within accepted practice?
- If using MCP or similar adapters, is access scoped and appropriate?

## Judgement
- Does this require human judgement beyond drafting or structuring?
- Is there any legal, financial, policy, service, or reputational implication?

## Review
- Has the output been reviewed?
- Has it been verified appropriately?
- Are caveats visible?

## Approval
- Does a human need to approve before action or release?

If any answer raises material concern, pause and escalate rather than improvising.
