# Professional Overlay

## Purpose

This overlay adapts the core framework for professional work where:
- outputs may affect others
- quality needs explanation
- decisions may need traceability
- documentation matters
- some work should be safer by design

## What belongs here

- structured task briefs
- explicit scope and assumptions
- review notes
- verification notes
- approval boundaries
- safer use rules
- approved tooling patterns
- scoped capability access
- proportionate record keeping

## What does not belong here

- unmanaged experimentation
- hidden tool usage
- unrestricted access patterns
- relying on memory alone for important work
- invisible automation of sensitive actions

## Professional operating stance

Use the framework to:
- improve quality and consistency
- make work easier to explain
- create clearer handovers
- reduce ambiguity
- create bounded repeatability

## Review expectations

Professional outputs should usually show:
- the task brief
- the chosen approach
- the assumptions
- the review outcome
- the verification outcome
- any caveats or escalation points

## Approval principle

Not all work needs formal sign-off.

But where work affects:
- policy
- money
- people
- service outcomes
- sensitive data
- reputation

a named human approval point should exist.

## MCP note

In professional use, MCP should be treated as a governed capability layer.

Use rules:
- only use approved capability patterns
- scope access to the minimum needed
- document repeatable usages
- do not rely on unmanaged access to systems or data

## Practical default

For most professional tasks:
- clarify
- classify
- plan
- gather context
- execute
- review
- verify
- approve if needed
- deliver
- capture learning
